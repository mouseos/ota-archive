# ota-archive

Android OTA archive: collected device `getprop` + resolved firmware metadata,
served to the app via **cdn.jsdelivr.net**. See the design in
[android-ota-finder/docs/server.md](https://github.com/mouseos/android-ota-finder/blob/main/docs/server.md).

## Layout
```
OTA/Google/{brand}/{model}/
  timestamps.json                 # [utc, ...]  (regenerated by Actions)
  index.json                      # [{ts, hasGetprop, hasFirmware, archived}]
  {ro.build.date.utc}/getprop.json    # submitted by the app via the Worker
  {post-timestamp}/firmware.json      # resolved by Actions (chain + Range metadata)
```

## Read URLs (app uses these; no Worker, no auth)
```
https://cdn.jsdelivr.net/gh/mouseos/ota-archive@main/OTA/Google/{brand}/{model}/timestamps.json
https://cdn.jsdelivr.net/gh/mouseos/ota-archive@main/OTA/Google/{brand}/{model}/{ts}/firmware.json
```

## Components
- `worker/` — Cloudflare Worker, write-only `POST /v1/getprop` (commits getprop.json).
- `scripts/` — Python run by Actions: `regen_timestamps.py`, `resolve.py` (checkin
  chain with ro.build.date.utc + ro.product.device fallback, HTTP-Range metadata),
  `list_pending.py`, `ia_upload.py` (archive.org). `checkin*.py` / `*_pb2.py` are the
  android-checkin reference protobufs.
- `.github/workflows/pipeline.yml` — cron (6h): regen → resolve → commit → purge jsDelivr;
  then a matrix job uploads pending firmware to archive.org.

## Setup (manual steps — credentials)

### 1. Cloudflare Worker (write API)
```bash
cd worker
npm install
npx wrangler login                       # browser auth (human)
npx wrangler secret put GH_TOKEN         # paste a fine-grained PAT (Contents: R/W on this repo)
# optional rate limit:
#   npx wrangler kv namespace create RL  # then uncomment kv_namespaces in wrangler.toml
npx wrangler deploy                      # -> https://ota-getprop.<sub>.workers.dev/v1/getprop
```
Put that endpoint URL into the app.

### 2. GitHub Actions secrets (this repo → Settings → Secrets → Actions)
- `IA_ACCESS_KEY`, `IA_SECRET_KEY` — from https://archive.org/account/s3.php
- (`GITHUB_TOKEN` is automatic; the workflow already has `permissions: contents: write`.)

### 3. archive.org
Create an account; generate S3 keys (above). Item id scheme:
`ota-google-{brand}-{model}-{postTimestamp}`.

### 4. App
- Set the Worker endpoint URL.
- Read timestamps.json / firmware.json from the jsDelivr URLs above.

No credentials are needed for Google checkin (anonymous) or jsDelivr.
