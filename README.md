# ota-archive

Android OTA archive: collected device `getprop` + resolved firmware metadata,
served to the app via **cdn.jsdelivr.net**. See the design in
[android-ota-finder/docs/server.md](https://github.com/mouseos/android-ota-finder/blob/main/docs/server.md).

## Layout
Paths are lowercased; `{group}` = `ro.product.manufacturer` (falling back to
`ro.product.brand`), so e.g. SBM801FJ lives under `sharp/`, not `sbm/`.
```
OTA/Google/{group}/{model}/
  index.json                      # {firmware:[...], builds:[...]} (regenerated by Actions)
  getprop/{sha1(fingerprint)}.json    # submitted by the app via the Worker
  firmware/{sha1(postBuild)}.json     # resolved by Actions (one per OTA)
```
The chain links by **fingerprint**, the only id present in both the device props
(`ro.build.fingerprint`) and OTA metadata (`pre-build`/`post-build`) — so it works
even when `ro.build.date.utc` is missing. A firmware's identity is its **postBuild**
(the build it installs); `preBuild` is just the build it updates from. Each firmware
records `otaType` (`full` | `incremental`, from metadata `pre-build` presence).

## Read URLs (app uses these; no Worker, no auth)
```
https://cdn.jsdelivr.net/gh/mouseos/ota-archive@main/OTA/Google/{group}/{model}/index.json
```
The app fetches this once and walks the chain in memory: match the device fingerprint
against `firmware[].preBuild`, follow `postBuild` → next.

## Components
- `worker/` — Cloudflare Worker, write-only `POST /v1/getprop` (commits getprop by
  fingerprint hash under the manufacturer group).
- `scripts/` — Python run by Actions: `resolve.py` (fingerprint-linked checkin chain
  with ro.product.device fallback + HTTP-Range metadata, writes firmware by postBuild
  hash + otaType), `regen_index.py` (per-model index.json), `list_pending.py`,
  `ia_upload.py` (archive.org). `checkin*.py` / `*_pb2.py` are the android-checkin
  reference protobufs.
- `.github/workflows/pipeline.yml` — cron (6h): resolve → regen index → commit → purge
  jsDelivr; then a matrix job uploads pending firmware to archive.org.

## Setup (manual steps — credentials)

### 1. Cloudflare Worker (write API)
```bash
cd worker
npm install
npx wrangler login                       # browser auth (human)
npx wrangler secret put GH_TOKEN         # paste a fine-grained PAT (Contents: R/W on this repo)
# optional rate limit:
#   npx wrangler kv namespace create RL  # then uncomment kv_namespaces in wrangler.toml
npx wrangler deploy                      # -> https://ota-getprop.<sub>.workers.dev/v1/getprop
```
Put that endpoint URL into the app.

### 2. GitHub Actions secrets (this repo → Settings → Secrets → Actions)
- `IA_ACCESS_KEY`, `IA_SECRET_KEY` — from https://archive.org/account/s3.php
- (`GITHUB_TOKEN` is automatic; the workflow already has `permissions: contents: write`.)

### 3. archive.org
Create an account; generate S3 keys (above). One item per device —
`ota-google-{group}-{model}` — with each firmware stored inside under its
original package filename.

### 4. App
- Set the Worker endpoint URL.
- Read timestamps.json / firmware.json from the jsDelivr URLs above.

No credentials are needed for Google checkin (anonymous) or jsDelivr.
